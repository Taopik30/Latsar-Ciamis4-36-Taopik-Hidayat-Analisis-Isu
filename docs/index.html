Analisis Akar Masalah (Fishbone)

Environment

  • Internet/ISP fluktuatif.
  • Kapasitas server kurang.
  • Lonjakan serentak.
  • Listrik tidak stabil.

Monitoring

  • Uptime tidak terukur.
  • Log tidak terstruktur.
  • Threshold alert keliru.
  • Tak ada dashboard real-time.

Konfigurasi

  • Query tanpa index.
  • DB tidak optimal.
  • Asset besar.
  • Library rentan.

Infrastruktur

  • Firewall L3/L4 saja (WAF L7 belum).
  • DB tunggal.
  • Single server.
  • CPU/RAM kurang, I/O bottleneck.

Proses

  • Tanpa load/stress test.
  • SOP incident belum baku.
  • Change tanpa review/rollback.
  • Backup tidak terjadwal.

Manpower

  • Skill security terbatas.
  • Ketergantungan vendor.
  • Higiene kredensial lemah.

Efek: Down/Eror ‚Ä¢ Layanan lambat ‚Ä¢ Risiko kebocoran data.

Catatan Penetapan Core Issue

Core issue dipilih melalui AKPL, USG, dan tapisan berbobot (Dampak 40%, Urgensi 30%, Risiko 15%, Kemudahan 15%). Isu ‚ÄúAplikasi rawan diretas & sering error saat trafik tinggi‚Äù memperoleh skor tertinggi karena berdampak lintas layanan, berisiko tinggi, sekaligus berada dalam kewenangan unit untuk diperbaiki. Analisis fishbone menunjukkan akar masalah tersebar pada infrastruktur (WAF belum aktif, DB & server tunggal), proses (tanpa load test, SOP incident belum baku), konfigurasi (query/index), monitoring (uptime & log belum terukur), SDM (keterampilan keamanan), dan faktor lingkungan (lonjakan serentak/ISP).

Solusi kreatif yang diajukan mencakup migrasi & hardening, aktivasi WAF‚ÄìTLS‚ÄìMFA, optimasi DB & caching, monitoring real-time, SOP incident & change, pembentukan tim incident response, serta penetapan IKU keamanan & uptime.
